总结

上上周六，也就是10月28号，我积压已久的情绪，和黑暗，还有所有的负能量一并爆发了出来，具体过程就不在过多的说明， 但是，的确是伤害了自己，也伤害了别人。我感到很自责，也很愧疚啊。
我觉得吧，所有的事情都是两面性的，那次爆发之后的第二天，我回到自己的出租屋里，也想了很多，大概是因为情绪得以宣泄，所以很多事情也想明白了，很多事情也不再纠结了。这反而可以让我以一个平静的心态去思考，的确，我那天思考了很多，也终于解决了自己的心结。而且那个时候刚好我接触到了冥想与正念这个东西，我觉得，对于那个时候的自己，可能缺少的就是平静，所以我买了一本关于冥想正念的书，照着书上的做，这也的确给我带来了一点益处。最大的益处就是，心绪平静。
那时候总结了一下自己当时的生活状态和心理状态，我觉得是压力太大了，所以那个时候我就想着给自己放假半个月，除了必要的工作之外，剩下的什么都不管，什么都不顾，只做自己喜欢做的。然而，我的确是做到了，今天也就是11月12号，是半个月假期的最后一天，所以我来做个汇报。

understanding and changing

发表于 2017-10-22

昨天我和一位女生一起看了电影，吃了下午茶，我不知道我现在该如何准确的定位我和这位女生的关系，但是大体说来就是她让我有了原本我以为我这辈子都不可能有的那种「不顾一切的冲动」，那种不顾一切的想要对一个人好的冲动，能遇到，就是一种幸运吧。

她

2017/6/26“hi～我也是乌兰察布的”。“哇～竟然遇到老乡了”2017/8/10“嗨，我是薛震”。“你和我想的不一样，我以为是瘦瘦的”。这就是我们第一次微信的对话，以及第一次面对面说话的内容。然后时间到了昨天，我们下午茶之后回家的路上谈了很多，我以为我有些话是绝对不会亲口说出来的，但是，就是这么神奇，说了很多矫情，但是又掏心窝子的话。但是呢，转念一想，难道这些日子发生的不可预料的事情还少吗？

understanding and changing

understanding

我能听懂但是不理解

这从字面意思上来说就是很直白的，“我知道你做了什么，但是我不知道你为什么这样做，我也不能理解你在做这些的时候是什么想法”。诚然，每一个人都有自己的想法，当这种想法强烈到一定的程度的时候，大多数人都会选择把这种想法付诸实际，实际这方面，很好理解，就是你看到了什么，那什么就是实际。而想法却很难理解，因为一眼看穿别人内心的机器还没发明出来。这个标题是我们昨天各自回家之后的微信聊天记录的一句，我知道她听懂了我和她说的字面意思，也知道她不理解的是我和她说的两个不，我想了一下这个问题，可能不理解的最根本原因就是不相信，我知道她当时可能并不相信我说的话。因为不相信，所以觉的说出的话，做的事感觉到匪夷所思的同事，也就觉得不能理解这样说/做的动机。而不相信，我觉得是有两个原因，第一 ，之前从来没有遇到过类似的情况，所以还不能接受。第二种就是完全的不信，就像是“地球是一个正方体”。不过我觉得，我应该是属于前者。对于前者这种情况我想说的是，人类大概是这个世界上心理最复杂的动物，每一个人所经历过的事情大体上都是不一样的，可能你觉得不能理解的事情，在对方看来就是很正常的事情，就是对方内心真实想表达的东西。倒是这些东西，不需要很快的被理解，只需要在平时慢慢的感受，如果真的是想理解的话，相信终有一天，会理解的。

changing

改变，也是昨天聊天中的一个小话题吧，没错，每一个人都需要改变，但是同时也需要坚持，重要的是，要改变什么和要坚持什么，就拿我来说，我需要改变的无非不是一些自己的消极情绪，以及消除自身的负能量，心向阳光。而我需要坚持的就是保持对自己爱的人，或者是事无穷尽的热情。可能有时候我们会因为之前的一些经历在大脑中形成一种固定的想法，或者说是标准，这种想法/标准想要改变的话很难，但是重要的，是你觉得自己的这种想法应不应该改变，以及你愿不愿意改变。
我觉得这个时候 pink 的“fucking perfect”里面的一句歌词还挺合适的：change the voices in ur head.


其实，在追求自己想要的同时，做出一些改变，或许会更好。自己内心的想法在潜意识里肯定是被自己认为是对的事情，不然，这种想法可能根本不会存在。我不知道改变，会对自己带来什么，每一个不同的人都会有不同的答案，比如说，这种“对”，让我很享受，我坚持 ，我会乐在其中，改变这种“对”，会让我变得痛苦，那么，最好不要改变。如果说这种“对”其实并没有给我带来什么实际意义上的东西，反而让我失去了很多本应该有的快乐，改变这种“对”，也许会让我感觉快乐，那么你就应该好好想想，你究竟是要“对”，还是要“快乐”。
这些道理说起来大家都懂，我知道做起来却并不尽然，你知道我说的有些道理，你说你会试着慢慢改变，如果你愿意给我这个机会，我愿意等着你的改变，愿意帮助你改变，愿意陪着你改变。以后的事情，就交给以后了，我只知道我现在坚持的事情是对的，这种对，让我快乐，所以我不会改变。

最后

昨天真的可能是自从八月份以来过的为数不多开心的日子里最开心的一天了，看了很棒的电影，吃了很棒的下午茶，但是我觉得昨天最大的收获，并不在于电影和下午茶，而是在于了解，谢谢你给我机会了解你，也谢谢你可以接受我这种没有套路笨拙的方式。你说：“我听懂了，但是不理解”。等你什么时候理解了，你就真的改变了。希望有一天，她会改变，希望有一天，你会改变。and i will always be here.

关于最近有关于ss可以被探测以及相关论文的一点看法

发表于 2017-10-12

起因

最近出现一些消息说ss可以被探测，且正确率可以达到75%之多，这项技术是由阿里云和360联合开发的，还写了论文，作为一个爱国人士，我迫不及待的看了一下这篇论文，看看是不是我们国家的科技水平发展的这么迅速，是不是我们都落后了。

经过

没有。是的，没有经过，经过就是我看了一下那篇论文。

正文

最近有两篇号称是检测 Tor 和 Shadowsocks 的论文。四个字，纯属放屁，当毕业设计都不一定及格。大家不用在上面浪费时间了。先不说论文作者认为「Shadowsocks 是用 SSH 连接的」，那论文连单词都拼不对（据说这已经是重写过的）。建议作者回到高中重新学习英文，或者给我 50 块钱让我代译英文版也行 [doge]

说道滥用问题，作者说：

To effectively prevent these things happened

「prevent things happened」，要么是作者重新发明了英文语法，要么是作者发明了时间机器。

提到使用代理访问新闻的现象，作者说：

the news even contain sensitive information in politics, economics, democratic, financial, technology and so forth.

先不说「contain」这个动词的问题，「politics」和「economics」还说得过去，都是名词。但「democratic」和「financial」是什么鬼，怎么突然变形容词了，这是什么语言？还「so forth」，什么鬼？

还说：

more and more people has learnt

看来不仅应该回到高中，还应该先回初中。

提到准确率的问题，作者说：

test set and feature set to reduce the false alarm rate and false rate when detecting

「false alarm rate and false rate」是什么鬼？这 false rate 好像说了两次？这体现了论文作者的最高水平——根本不知道 false-positive 和 false-negative 两个单词，统计学估计是不及格的。

说道检测的重要性，作者说：

do the following decrypted work and content analysis.
「随后进行解密工作」变成了「do decrypted work」，我还「encrypted work」呢……听说论文作者是研究网络安全的，听没听说过一种非常尖端的科技，叫做这个——「cryptoanalysis」？

唯一没什么问题的，是这半句话：

and to immediately detect and arrest these criminals

好大的口气！「immediately detect and arrest criminals!」「immediately! 」看来说的是真心话，语法一点问题没有。

以及：

Shadowsocks make proxy by using established channel, which means that requesting the real servers by SSH server. Lastly, Servers through SSH server send back the responding data by using established channel. Since SSH itself is based on RSA……

说 Shadowsocks 是通过 SSH 连接的……作者可能分不清「通过 SSH 进行网络连接」以及「装了一个 SSH 能管理系统」的服务器有什么区别。或者说……难道作者这篇论文是故意放水的，发扬了枪口抬高一光年的精神，真是令人肃然起敬。

这两篇文本的 DOI 编号分别为：

10.1109/IHMSC.2017.132
10.1109/IHMSC.2017.133

大家可以去 sci-hub.cc 输入 DOI 直接下载盗版 PDF，打印出来，挂在你的学校、办公室或者实验室，提醒自己和同行：如果不加倍努力，将来你的水平就会像这篇论文一样。

Hello World

发表于 2017-09-23

关于我

今天终于把自己的博客 Snow’s blog搭建好并且上线了! 起先，就是因为工作的问题，我的心态有点不好，所以我觉得我应该记录一些自己的生活，写点什么东西以后留给自己。因为微博太霸道了，生怕哪天一个不小心被哪位老大哥盯上，账号被注销，Twitter呢又不能发长推。所以之前就在Telegram开了一个私人的channel但是后来发现会出现记录丢失的情况，而且不好管理，所以才动了搭建博客的想法。我搭建这个博客只是为了记录自己生活上的一些事情，比如工作、学习上的感悟，也有可能会出现技术相关的东西。当然也会有吐槽，也会有负能量，毕竟这个博客并不是为了阅读量而生的。仅仅只是我自己和自己玩儿，以及记录生活所用。

关于这个博客

这个博客是我在macos平台上利用hexo部署在github上面的，目前的情况就是基本上处于一个可以上线的状态，但是也仅仅只是可以上线，有好多需要配置的东西还没有配置，好多需要学习的东西还没有学习。目前来说还是比较简陋的，以后我会尽量完善一些东西。

关于坑

说起来，建站这件事情，是我之前从来没接触过的，当自己有搭建博客的这个想法开始，我自以为是的觉得还是很简单的，毕竟“艺高人胆大”觉得凭着自己的计算机水平做个博客也就是一两个晚上的事情，于是乎一鼓作气，买了域名和vps。结果呢，我发现，果然，在建站的所有步骤中，只有花钱是最简单的，毕竟只需要用手机扫一下，输入一个密码。然后，第一天实际开始的时候，现实就给了我一记响亮的耳光，windows环境下各种环境变量的配置就给我上了一课，第一天晚上，我记得甚至我连环境都没装好，我还记得，我那个时候就想着放弃算了，后来，好在我坚持了下来，通过网上查找各种资料，环境终于是弄好了，没想到接下来更深的坑在等着我：如何把本地写好的博文部署到我自己的vps上，开始时候由于密钥问题，不能登录到vps，接下密钥问题解决了，终于可以登陆vps的时候，我发现vps操作系统是Linux，而我仅有的一点Linux知识也完全是不够用的，之后就是漫长的查找资料的过程。好不容易服务器断搞定了，又因为万恶的环境变量配置问题导致了不能生成静态网页，于是乎，接近于前功尽弃，再后来，环境变量解决了，远程部署上又出了问题，总之就是各种问题，感觉自己把做这个工作所有的坑都踩了一次。总结一下就是自己的知识水平不够，所以出了问题就只能网上找解决办法，由于这种工作效率的低下，也导致了我今天才把所有问题都解决掉。

关于今后

今后会不定期更新，内容也是会千奇百怪，总体还是以自己的生活感悟还有技术方面为主，但是总体的涉及领域也会有很多，比如我熟悉的计算机相关领域，科技数码相关领域。也有我不熟悉的设计领域，音乐领域等。还有我比较感兴趣的历史相关。我觉得大概我是一个无趣的人吧，自己生活在自己的那一小片地方，闭门造车，固步自封，不愿意接受一些别的方面的东西，虽然算不得是井底之蛙，但是我感觉我的确是错过了一些有趣的东西。所以自己今后准备多了解一些新的领域的知识。

最后

希望自己会越来越好，努力成为一个更好的人。就是这样
